    Step 1: SME (Responsible Engineer or RE) creates and submits a TPT ticket to MarCom (TPT is the technical publication tracking system used to track documentation tickets, and is launched in sharepoint site)
    Step 2: MarCom Reviews Ticket and accepts the ownership
    Step 3: Marcom downloads the source information which is generally attached to the ticket as a word document or a marked PDF file. 
    Step 4: Marcom checks if any clarification is needed. If needed, connects with the SME
    Step 5: If no clarification is needed, MarcCom creates a new publication (for new release notes) or opens a new version of an existing publication (for release note updates) in Tridion Docs
    Step 6: MarCom makes the necessary changes in the respective topic in DITA
    Step 7: Generates the updated document output in Tridion Docs 
    Step 8: Creates a Adobe shared review and shares it with all the reviewers mentioned in the TPT
    Step 9: Iterates the process until all the SMEs/reviewers confirm the final draft
    Step 10: Final draft is released in the Tridion Docs
    Step 11: Release note is made public in the website through AEM
