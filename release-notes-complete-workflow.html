Workflow Stages

1. Generate Extract information from source material and create initial release note content Release Note Specialist, Documentation Style Guide Draft release notes with customer-focused language

Key Principles Applied

  • Accuracy First: Use only information provided in source materials; never invent details
  • Customer-Focused Writing: Emphasize user outcomes and benefits rather than implementation details
  • MSTP Compliance: Use active voice, present tense, and clear, concise language
  • Consistency: Maintain consistent terminology and formatting throughout
  • Clarity: Avoid jargon and ambiguous statements
  • Information Gaps: Explicitly identify missing information that would strengthen the release notes

Quality Checkpoints

  • All claims validated against source information
  • Customer focus confirmed in messaging
  • Terminology consistency verified
  • Grammar and spelling checked
  • MSTP principles applied throughout
  • Missing information documented

Workflow Benefits

  • Time Efficiency: AI-assisted generation reduces manual drafting time
  • Consistency: Skills and style guide ensure uniform quality across releases
  • Quality Improvement: Structured review process identifies gaps and improvement opportunities
  • Risk Reduction: Multiple review stages catch inaccuracies and missing information
  • Scalability: Process handles multiple scenarios and release types consistently
