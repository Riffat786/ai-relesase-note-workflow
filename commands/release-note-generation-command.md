# Generate Release Notes

## Purpose

Generate customer-facing release notes from provided source information.

## Instructions

Use the following skills:

- Documentation Style Guide
- Release Note Specialist

Analyze the provided source information and create review-ready release notes.

Follow these requirements:

1. Use only information provided.
2. Do not invent missing details.
3. Use customer-focused language.
4. Follow MSTP principles.
5. Classify content appropriately.
6. Identify missing information.
7. Exclude implementation details unless specifically requested.
