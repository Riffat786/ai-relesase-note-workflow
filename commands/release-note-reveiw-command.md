# Review Release Notes

## Purpose

Review release note content for quality, completeness, consistency, and compliance with documentation standards.

## Instructions

Use:

- Documentation Style Guide
- Documentation Reviewer

Perform a comprehensive review.

Validate:

- Accuracy
- Clarity
- Consistency
- Customer focus
- Terminology
- MSTP compliance
- Missing information

Do not rewrite content.

Provide findings and recommendations.

## Input

Draft Release Notes

## Output Structure

# Review Result

PASS
PASS WITH RECOMMENDATIONS
FAIL

# Findings

# Recommendations

# Risks

# Missing Information

# Open Questions
