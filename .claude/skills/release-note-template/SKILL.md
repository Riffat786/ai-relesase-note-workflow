# Release Note Template

## Purpose

Apply the organization's approved release note template. 

Define the standard structure and formatting for all release notes.

## Required Sections

- Release Title
- Overview
- New Features
- Enhancements
- Bug Fixes
- Recommended Actions
- Known Limitations

## Formatting

- Markdown
- Consistent headings
- Tables where appropriate
- Professional formatting
- Preserve internal work item IDs

---

## Required Structure

```markdown
# Release <Version>

## Overview

Provide a short summary of the release.

## New Features

Describe new customer-facing capabilities.

## Enhancements

Describe improvements to existing functionality.

## Bug Fixes

Describe resolved customer-facing issues.

## Known Issues (Optional)

List known limitations or outstanding issues.
```

---

## Formatting Rules

* Use Markdown headings.
* Create a table for each type with three columns - ID, title and description
* Use sentence case for headings.
* Keep feature descriptions to one or two paragraphs.
* Use bullet lists only when appropriate.

---

## Writing Order

1. Overview
2. New Features
3. Enhancements
4. Bug Fixes
5. Known Issues

---

## Success Criteria

Every release note:

* Follows the same structure.
* Is easy to scan.
* Is consistent across releases.
