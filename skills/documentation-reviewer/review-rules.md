# Review Rules

## Ambiguity Detection

Flag vague statements.

Examples:

- Improved performance
- Enhanced functionality
- Better user experience

Unless supported by specific details.

---

## Unsupported Claims

Flag statements such as:

- Faster
- Better
- Improved
- More secure

Unless evidence is provided.

---

## Missing Information

Identify gaps such as:

- Customer impact not provided
- User audience unclear
- Feature description incomplete

---

## Terminology Compliance

Verify approved terminology is used.

Reference:

terminology.md

---

## Style Compliance

Verify alignment with:

Documentation Style Guide Skill

Including:

- Active voice
- Present tense
- Customer focus
- Conciseness
