# Classification Rules

## Purpose

Assign changes to the appropriate release note section.

---

# Classification Mapping

| Issue Type | Release Note Section |
|------------|----------------------|
| Feature | New Features |
| Enhancement | Enhancements |
| Bug | Resolved Issues |
| Security | Security Updates |
| Limitation | Known Limitations |

---

# Customer Visibility Assessment

Include:

- New capabilities
- Customer-facing improvements
- User-impacting fixes
- Security improvements

Review Before Including:

- Configuration changes
- Infrastructure updates
- Performance changes without user impact

Exclude Unless Requested:

- Refactoring
- Internal tooling
- Development process updates
- Test updates

---

# Ambiguous Classification

If classification is unclear:

Provide a recommendation and explain the reasoning.
