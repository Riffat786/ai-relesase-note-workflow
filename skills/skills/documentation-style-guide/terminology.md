# Terminology Standards

## Purpose

This document defines approved terminology and naming conventions to ensure consistency across all documentation.

---

# General Rules

- Use official product names.
- Use official feature names.
- Do not invent abbreviations.
- Do not create alternate names.
- Use terminology consistently throughout a document.

---

# User References

Preferred:

- User
- Users
- Administrator
- Administrators
- Customer
- Customers

Avoid:

- End User
- Consumer

Unless organizational standards require them.

---

# Documentation Terminology

Preferred:

| Use | Avoid |
|------|------|
| Sign in | Log in (unless product standard) |
| Select | Click |
| Choose | Pick |
| Configure | Setup |
| Information | Data (when user-facing) |

---

# Inclusive Language

Use:

- Primary
- Secondary
- Main
- Subordinate
- Allowlist
- Blocklist

Avoid outdated or non-inclusive terminology.
